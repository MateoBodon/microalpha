## Repo Mission

microalpha exists to become a portfolio-grade quantitative research project:
a serious, leakage-safe, reproducible empirical research engine with the
strongest defensible real-data WRDS/CRSP results the project can achieve.

## Long-Term Goal

Build an impressive, prestigious, externally presentable quant research project
with:

- credible real-data WRDS/CRSP results;
- transparent walk-forward and holdout methodology;
- evidence-backed performance claims;
- leakage, survivorship, cost, and overfitting controls;
- a concise public portfolio narrative.

## Near-Term Goal

Run a WRDS prestige evidence phase:

1. install this Pro strategy package as current truth;
2. recover and inventory data/artifacts after the server transfer;
3. reproduce or explicitly block the current WRDS baseline;
4. expand WRDS data if possible, preferably beyond the already-used 2018-2019 holdout;
5. run a high-ambition, evidence-disciplined WRDS research campaign.

## Priority Order

1. Prestigious real-data performance and research credibility.
2. Evidence discipline as a hard gate for prestige.
3. Ambitious throughput through large coherent Codex tickets.
4. Portfolio/recruiting presentation after evidence is strong.
5. Minimal necessary polish.
6. Process maintenance only when it accelerates future work.

## Non-Goals

- Do not become a live trading/brokerage system.
- Do not optimize public mini-panel results as the main prestige path.
- Do not spend the next phase on broad documentation cleanup.
- Do not make alpha/performance claims without artifact-backed L3/L4 evidence.
- Do not commit, bundle, or expose raw WRDS/CRSP data.
- Do not split ambitious research work into tiny conservative tickets.

## Quality Bar

Every outward-facing result claim needs run ID, git SHA, config hash, dataset ID
or data manifest, train/selection/holdout windows, costs/slippage/borrow
assumptions, non-degenerate trade evidence, baselines, statistical caveats, and
validation commands.

## What Future Sessions Must Not Forget

- User wants maximal prestige/performance, not conservative process drift.
- Public mini-panel evidence is currently degenerate pipeline evidence only.
- WRDS evidence is the main prestige path but claim-sensitive and local-data-dependent.
- The 2018-2019 WRDS holdout has been used repeatedly; a new prestige campaign
  should prefer a fresher final holdout if expanded data is available.
- Raw WRDS data is local-only.
- The T-000 audit bundle indexed current evidence paths but did not include exact
  current result artifact contents, so Heavy/Codex must verify them before public
  claim wording.
