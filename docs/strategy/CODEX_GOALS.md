## Current Codex Frontier

Codex should help move microalpha into a WRDS Prestige Evidence Phase.

## Highest-Value Work

1. Install Pro strategy docs and stabilize current evidence state.
2. Verify actual current result artifacts and claim surfaces.
3. Fix or neutralize Python import drift so tests validate the local checkout.
4. Inventory post-transfer data/artifacts without exposing raw WRDS data.
5. Reproduce or explicitly block the current WRDS baseline.
6. Prepare or execute expanded WRDS data export/manifest work.
7. Run a high-ambition WRDS research campaign with pre-registered selection rules,
   baselines, non-degenerate checks, and leaderboard outputs.
8. Build public/portfolio claim surfaces only after evidence is approved.

## Codex Must Not

- Commit or bundle raw WRDS/CRSP data.
- Turn public mini-panel output into performance claims.
- Use old docs as current truth without canonical migration.
- Claim alpha, superiority, or statistical significance without evidence.
- Run broad strategy search without recording protocol, configs, and holdout rules.
- Spend a sprint on cosmetic docs unless claim safety requires it.

## Default Validation Expectations

- L0: `git status --short`.
- L1: targeted syntax/lint/script checks for touched tooling.
- L2: `make check-data-policy`, local-source fast tests, docs build when docs touched.
- L3: WRDS pipeline reproduction when data is available.
- L4: claim/evidence audit before public/recruiting use.
